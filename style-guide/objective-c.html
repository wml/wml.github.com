Style Guide - Objective-C

Color  •  Objective-C

Spacing __

  • Use four spaces to indent blocks; never use tabs
  • Include a blank line between function definitions
  • Put a single space after the keywords if, while, and for
  • Put a single space after colons, closing parentheses, and argument names in method signatures and invocations
  • Don't put a space between the type and pointer specifier (*) in method signatures
  • Do put a space between the type and pointer specifier (*) when declaring variables, but don't include a space before the variable name

Braces {}

  • Put opening braces on the same line
  • Put closing braces on their own line
  • Always use braces around if, while, and for bodies

Properties @@

  • Don't use any property modifiers except nonatomic
  • Don't explicitly @synthesize properties
  • Don't expose unnecessary properties where fields will suffice

Naming id

  • Name fields with a leading underscore as with auto-synthesized properties
  • Name methods such that they read approximately as English
  • Local variables and methods should be lowerCamelCased
  • Classes, block typedefs and constants should be UpperCamelCased
  • Don't use abbreviations unless they're universally used in place of the full word
  • Only upper-case the first letter of an acronym (SslHttpUrl vs. SSLHTTPURL)
  • Don't use self to refer to fields unless necessary

Lines & Alignment \n

  • Wrap lines longer than 120 characters
  • When method signatures or invocations wrap lines, align method name tokens to the same column

Code Organization ./

  • Limit Xcode groups to logical high-level components
  • Place header files in include, implementations in lib, and non-source assets in resources
